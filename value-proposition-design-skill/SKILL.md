# Value Proposition Design Skill

基于《价值主张设计：如何构建商业模式最重要的环节》（亚历山大·奥斯特瓦德著）的完整方法论技能。本Skill不仅提供知识参考，更具备6大可执行模块，能够针对具体业务场景直接产出结构化交付物。

## 适用场景

当用户需要进行以下工作时，调用此技能：
- 设计或评估价值主张
- 构建或优化商业模式
- 进行客户洞察和用户研究
- 设计创新方案并进行验证测试
- 做业务决策和竞争分析
- 进行量化实验和A/B测试设计

## 核心框架

### 价值主张画布（Value Proposition Canvas）

价值主张画布是核心工具，由两面构成：

**右侧：客户概况（Customer Profile）**
- 客户工作（Jobs）：功能性工作、社会工作、个人/情感工作、支持性工作（价值购买方、共同创造者、转移者）
- 客户痛点（Pains）：不想要的结果/问题/特性、障碍、风险。需量化具体化，如"等待超过X分钟"
- 客户收益（Gains）：必需的、期望的、渴望的、意外的。需具体化，如"提升到超越X"

**左侧：价值图（Value Map）**
- 产品和服务：有形的、无形的、数字的、财务的
- 痛点缓释方案：描述如何减轻特定客户痛点
- 收益创造方案：描述如何创造客户收益

### 三种契合类型

1. **问题-方案契合**（书面上）：有依据表明客户关注的工作/痛点/收益，并设计了对应价值主张
2. **产品-市场契合**（市场上）：有证据表明产品和服务能真正创造客户价值并受市场欢迎
3. **商业模式契合**（银行里）：有证据表明价值主张能植入可盈利和可实现的商业模式

## 执行模块一：访谈提纲生成器

### 触发条件

当用户提出以下类型需求时激活本模块：
- "帮我生成一份访谈提纲"
- "我要做用户访谈，场景是……"
- "设计一个针对XX客户群的深度访谈"
- 任何涉及"访谈""采访""用户沟通"的需求

### 必需输入

| 参数 | 说明 | 示例 |
|------|------|------|
| 业务场景 | 当前产品/服务所处的业务领域 | "飞猪国内酒店预订" |
| 目标客户群 | 要访谈的客户细分 | "25-35岁商旅用户" |

### 可选输入

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 已知假设 | 已有的客户工作/痛点/收益假设 | 无 |
| 访谈时长 | 预计单次访谈时间 | 45分钟 |
| B2B/B2C | 客户类型 | B2C |
| 访谈阶段 | 探索期/验证期/优化期 | 探索期 |

### 执行流程

**第一步：确定访谈目标**

根据用户输入的业务场景和客户群，明确本次访谈的核心目标：
- 探索期：发现客户的工作、痛点和收益（开放式）
- 验证期：验证已有的价值主张假设（半结构化）
- 优化期：评估现有方案的契合度（结构化）

**第二步：生成访谈提纲**

严格按以下五阶段结构输出，每阶段标注建议时长：

```
# [业务场景] - [客户群] 访谈提纲
访谈目标：[一句话目标]
预计时长：[X]分钟
访谈人数建议：[N]人（建议至少10人以发现模式）

## 阶段一：暖场与背景（5分钟）
目的：建立信任，了解受访者基本情况

1. 请简单介绍一下您自己和您的[工作/生活]情况？
2. 您在[业务领域]方面的日常是怎样的？

## 阶段二：工作探索（10-15分钟）
目的：发现客户正在尝试完成的功能性、社会性和情感性工作
原则：问"做什么"和"为什么"，不问"想要什么"

### 功能性工作
3. 在[场景]中，您通常需要完成哪些事情？
4. 请描述一下您最近一次[完成某项工作]的完整过程？
5. 在这个过程中，哪些步骤是您必须亲自做的？

### 社会性工作
6. 在[场景]中，您希望在他人眼中呈现什么样的形象？
7. 谁的看法对您在这件事上的决策影响最大？

### 情感性工作
8. 做这件事时，您最希望获得什么样的感受？
9. 有没有什么让您感到安心或不安的因素？

### 支持性工作（如适用）
10. 您是如何找到和选择[产品/服务]的？
11. 购买后，您是如何开始使用的？遇到过什么困难吗？

---

## 阶段三：痛点挖掘（10-15分钟）
目的：发现客户在完成工作过程中的障碍、风险和不满
原则：追问具体事例，量化痛苦程度

12. 在[完成工作]的过程中，最让您头疼/沮丧的是什么？
13. 能具体描述一次让您特别不满意的经历吗？（追问：发生了什么？结果如何？）
14. 有什么因素曾经阻止您[完成某项工作]或让您放弃？
15. 在这个过程中，您最担心出现什么风险或错误？
16. 如果用1-10分衡量这个痛点的严重程度，您会打几分？为什么？
17. 目前您是怎么应对这些问题的？这些替代方案有什么不足？

---

## 阶段四：收益发现（10分钟）
目的：发现客户期望的结果和超出预期的惊喜
原则：区分必需收益、期望收益、渴望收益和意外收益

18. 对您来说，[完成这项工作]最理想的结果是什么？
19. 什么情况下您会觉得这次体验"超出预期"？
20. 如果有一个魔法棒可以改变[场景]中的一件事，您会改变什么？
21. 您见过哪些[竞品/替代方案]让您觉得"做得真好"的地方？具体好在哪里？
22. 在[场景]中，您愿意为什么样的改善额外付费？

---

## 阶段五：收尾与拓展（5分钟）
目的：捕获遗漏信息，获取转介绍

23. 关于[业务场景]，还有什么我们没聊到但您觉得重要的事情吗？
24. 在您认识的人中，谁也经常[做这项工作]？我可以和他们聊聊吗？
25. 如果我们后续有新的想法想请您看看，方便留个联系方式吗？

---

## 访谈执行要点

### 八大原则（必须遵守）
1. 采取初学者心态——假设自己什么都不知道
2. 多听少说——80%时间让受访者说
3. 求得事实而非观点——问"上次您怎么做的"而非"您觉得怎么样"
4. 问"为什么"探究动机——至少追问3层
5. 目的是学习而非推销——绝不介绍自己的产品
6. 不要太早提及解决方案——让客户自由表达
7. 跟进保留联系信息——为后续验证做准备
8. 总在最后留个门——"我还可以跟谁谈？"

### 执行建议
- 两人一组：一人引导对话，一人记录
- 记录方式：在空白客户概况框架上实时捕捉工作/痛点/收益
- 每完成5次访谈后回顾，调整提纲侧重点
- B2B场景需为每个角色（影响者/决策者/使用者/购买者）准备不同版本
```

**第三步：输出B2B多角色版本（如适用）**

如果是B2B场景，额外为以下角色各生成一份侧重不同的提纲：
- 最终用户：侧重功能性工作和日常痛点
- 经济购买者：侧重成本、ROI和预算约束
- 决策者：侧重战略价值和风险
- 影响者/建议者：侧重评估标准和行业趋势

### 输出格式

以Markdown格式输出完整访谈提纲，包含所有阶段的问题、执行要点和注意事项。如为B2B场景，输出多角色版本。

## 执行模块三：优先级计算模型

### 触发条件

当用户提出以下类型需求时激活本模块：
- "帮我判断哪些痛点/工作/收益最值得关注"
- "我有一组调研数据，帮我排优先级"
- "做一个优先级矩阵"
- 任何涉及"优先级""排序""打分""哪个更重要"的需求

### 必需输入

| 参数 | 说明 | 示例 |
|------|------|------|
| 评估对象列表 | 需要排序的工作/痛点/收益列表 | ["价格不透明","退改困难","客服响应慢"] |
| 评估维度 | 工作/痛点/收益（三选一或混合） | "痛点" |

### 可选输入

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 问卷/访谈数据 | 已有的定量或定性数据 | 无，由AI引导逐项打分 |
| 评分者 | 用户自评/团队评/客户数据 | 用户自评 |
| 竞品覆盖情况 | 竞争对手是否已解决各项 | 无 |

### 执行流程

**第一步：构建评估矩阵**

对每个评估对象，按以下4个维度进行1-5分打分：

| 维度 | 定义 | 打分标准 |
|------|------|---------|
| 重要性（I） | 该项对客户完成核心工作的重要程度 | 1=无关紧要，5=不可或缺 |
| 不满意度（D） | 客户对当前状况的不满程度 | 1=非常满意，5=极度不满 |
| 频率（F） | 客户遇到该项的频率 | 1=极少发生，5=每次都遇到 |
| 可行性（V） | 我方解决该项的技术和资源可行性 | 1=极难实现，5=立即可做 |

**第二步：计算优先级分数**

```
优先级分数 = 重要性(I) × 不满意度(D) × 频率(F) × 可行性(V) / 625 × 100

满分 = 5 × 5 × 5 × 5 = 625 → 归一化为100分
```

如果有竞品覆盖数据，增加差异化系数：
```
差异化系数(C) = 竞品未覆盖为1.5，部分覆盖为1.0，完全覆盖为0.5
最终分数 = 优先级分数 × 差异化系数(C)
```

**第三步：输出优先级矩阵**

按以下格式输出完整结果：

```
# [业务场景] 优先级评估矩阵

评估日期：[日期]
评估维度：[工作/痛点/收益]
评估对象数量：[N]项

## 评分明细

| 排名 | 评估对象 | 重要性(I) | 不满意度(D) | 频率(F) | 可行性(V) | 差异化(C) | 最终得分 | 优先级 |
|:----:|---------|:---------:|:----------:|:------:|:--------:|:--------:|:-------:|:-----:|
| 1 | [对象A] | 5 | 5 | 4 | 4 | 1.5 | 96.0 | P0 |
| 2 | [对象B] | 4 | 4 | 5 | 3 | 1.0 | 38.4 | P1 |
| 3 | [对象C] | 3 | 3 | 3 | 4 | 1.0 | 17.3 | P2 |
| ... | ... | ... | ... | ... | ... | ... | ... | ... |

## 优先级分级标准

| 等级 | 分数范围 | 建议行动 |
|:----:|:-------:|---------|
| P0 | ≥60分 | 立即行动：这是核心机会，应最优先投入资源 |
| P1 | 30-59分 | 重点关注：纳入近期规划，分配专项资源 |
| P2 | 15-29分 | 持续观察：纳入中期规划，等待时机 |
| P3 | <15分 | 暂时搁置：当前不值得投入，定期重新评估 |

## 决策建议

### 核心发现
[基于分数分布，总结1-3条核心发现]

### 建议聚焦
[基于P0和P1项目，给出具体的价值主张设计方向建议]

### 风险提示
[指出可能的假正面/假负面陷阱，建议进一步验证的方向]
```

**第四步：引导式打分（当用户无现成数据时）**

如果用户没有现成数据，逐项引导用户打分：
- 逐个列出评估对象
- 对每个对象依次询问4个维度的打分（1-5分）
- 实时计算并更新排名

## 模块联动说明

6大模块并非孤立使用，它们构成一个完整的价值主张设计工作流：

```
模块1（访谈提纲）→ 采集原始数据
        ↓
模块4（画布填充）→ 整理为结构化画布
        ↓

        ↓

        ↓

        ↓
模块6（实验设计）→ 设计MVP测试方案
        ↓
      迭代循环 → 回到模块1或模块4重新开始
```

当用户的需求涉及多个模块时，按上述工作流顺序依次执行，每个模块的输出作为下一个模块的输入。

## Python 可执行工具包（vpd）

本Skill附带完整的Python计算工具包 `vpd/`，提供精确计算能力。当AI需要执行数值计算、生成结构化报告时，应优先调用Python代码而非心算。

### 安装与依赖

```bash
pip install pyyaml
```

### 5大核心模块

#### 1. PriorityCalculator — 优先级计算

4维评分（重要性×不满意度×频率×可行性），归一化100分制，P0-P3自动分级，支持竞品差异化系数。

```python
from vpd import PriorityCalculator

calc = PriorityCalculator()
calc.add_item("价格不透明", importance=5, dissatisfaction=5, frequency=4, viability=4, competition_coverage="not_covered")
calc.add_item("退改困难", importance=4, dissatisfaction=4, frequency=3, viability=3)
calc.add_item("客服响应慢", importance=3, dissatisfaction=3, frequency=5, viability=5)
print(calc.render_markdown())
```

#### 2. CanvasAnalyzer — 价值主张画布分析

自动填充客户概况（工作/痛点/收益）和价值图（产品/缓释/创造），输出契合度评分和缺口分析。

```python
from vpd import CanvasAnalyzer

ca = CanvasAnalyzer()
ca.customer_name = "亲子家庭用户"
ca.product_name = "飞猪酒店预订"
ca.add_job("预订合适的家庭房", "functional", importance=5)
ca.add_pain("价格不透明", "critical", "比其他平台贵10%以上")
ca.add_gain("一键预订省时间", "required")
ca.add_product("智能比价引擎", "digital")
ca.add_pain_reliever("实时比价展示", "价格不透明", "full")
ca.add_gain_creator("一键下单功能", "一键预订省时间", "full")
print(ca.render_markdown())
```

#### 3. StrategyScorer — 竞争战略画布

竞争因素评分、文本版价值曲线、蓝海四项行动框架（剔除/减少/增加/创造）。

```python
from vpd import StrategyScorer

ss = StrategyScorer()
ss.set_factors(["价格", "品质", "速度", "服务", "个性化"])
ss.add_player("我方", [7, 8, 5, 6, 4])
ss.add_player("竞品A", [8, 6, 7, 5, 3])
ss.add_player("竞品B", [6, 7, 8, 4, 2])
print(ss.render_markdown("我方"))
```

#### 4. SampleCalculator — 样本量计算

基于统计显著性的最小样本量计算，支持有限总体修正。

```python
from vpd import SampleCalculator

sc = SampleCalculator()
print(sc.render_markdown(confidence=95, margin_of_error=0.05, population=1000))
```

#### 5. ExperimentDesigner — 实验设计

假设拆解排序、测试卡/学习卡生成、CTA分层、投资就绪水平评估。

```python
from vpd import ExperimentDesigner

ed = ExperimentDesigner()
ed.add_hypothesis("用户愿意为无忧退改额外付费30元", lethality="lethal")
ed.create_test_card(
    hypothesis="用户愿意为无忧退改额外付费30元",
    test_method="登录页MVP", metric="点击率",
    threshold="5%", falsification="点击率<2%则否定",
    cta_level="L3", duration_days=14, sample_size=200,
)
print(ed.render_markdown())
```

### 配置文件

`vpd/config.yaml` 包含所有可调参数：评分维度锚点、P0-P3阈值、竞品覆盖系数、画布契合度阈值、CTA层级定义、投资就绪水平量表、置信度Z值表等。可根据业务需要修改。

### AI Agent 调用规则

当用户需求涉及以下场景时，Agent应自动调用Python代码：
1. 优先级排序/打分 → `PriorityCalculator`
2. 画布填充/契合度分析 → `CanvasAnalyzer`
3. 竞品对比/差异化分析 → `StrategyScorer`
4. 样本量计算 → `SampleCalculator`
5. 实验方案设计 → `ExperimentDesigner`

调用流程：先通过对话收集用户输入，然后调用对应模块的方法计算，最后用 `render_markdown()` 生成结构化报告输出给用户。
