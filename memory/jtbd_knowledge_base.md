> 整理：小渡 🔍  
> 用途：用户研究、产品策略、创新分析

## 一、JTBD 核心定义

### 1.1 什么是 Job to be Done？

**定义 1（Christensen）**：
> A Job is the progress that an individual is trying to make in a particular circumstance.
> （Job 是个人在特定情境下试图取得的进步）

**定义 2（Klement）**：
> A Job to be Done is the process a consumer goes through whenever she aims to transform her existing life-situation into a preferred one, but cannot because constraints are stopping her.
> （JTBD 是消费者试图将现有生活情境转变为理想情境时所经历的过程，但因约束而无法实现）

### 1.2 关键理解

- **顾客不买产品，他们买更好的自己**：顾客想要的是生活情境的转变/进步，不是产品本身
- **产品是手段，进步是目的**：产品帮助顾客完成 Job，但产品本身不是 Job
- **只有人有 Job，事物没有**：不要说"产品的 Job 是..."，要说"顾客的 Job 是..."

### 1.3 什么不是 JTBD？

- ❌ 产品功能或属性
- ❌ 具体的任务或活动（这些是解决方案，不是 Job）
- ❌ 人口统计特征
- ❌ 顾客说的"需求"（往往是针对现有产品的抱怨）

## 二、JTBD 核心原则

### 2.1 八大原则

1. **顾客不想要你的产品，他们想要帮助自己变得更好**
   - 例子：Revlon 卖的不是化妆品，是"希望"
   - 例子：顾客买电钻不是为了要钻头，是为了墙上的洞（甚至可能不需要洞）

2. **人有 Job，事物没有**
   - 产品没有生命去改善，没有动机、抱负或挣扎
   - 只有人才有想要改善的生活

3. **竞争由顾客心智定义，进步是评判标准**
   - 竞争不是由产品功能或外观决定
   - 顾客用"能否帮助进步"来判断哪些产品互为竞争

4. **当顾客开始用一个解决方案，他们就停止使用其他方案**
   - JTBD 竞争是零和游戏
   - 一个顾客同一时间只偏好一个解决方案

5. **当顾客表现出补偿行为时，存在创新机会**
   - 例子：小苏打原本用于烘焙，顾客开始用作清洁剂和除臭剂
   - 例子：Segway 原定用于大众交通，却被执法部门和旅游公司使用

6. **偏好进步而非结果和目标**
   - 目标和结果只是行动的结果
   - 顾客在使用产品的每个触点都需要感受到进步

7. **进步定义价值，对比揭示价值**
   - 产品本身没有价值，只有在帮助顾客进步时才有价值
   - 价值通过对比不同情境下的进步来体现

8. **解决方案的价值超越使用时刻**
   - 价值不仅在使用时产生，还延伸到拥有产品的满足感
   - 例子：停在车库的车仍在提供价值（随时可用的安心感）

## 三、进步的四力模型（Forces of Progress）

### 3.1 四力概述

```
        需求生成 (Demand Generation)
        ┌─────────────────────────────┐
        │                             │
    ┌───▼───┐                     ┌───▼───┐
    │  Push │                     │  Pull │
    │  推力  │                     │  拉力  │
    └───┬───┘                     └───┬───┘
        │                             │
        │      ┌───────────┐          │
        │      │  顾客     │          │
        │      └───────────┘          │
        │                             │
    ┌───▼───┐                     ┌───┬───┐
    │Inertia│                     │Anxiety│
    │ 惯性  │                     │ 焦虑  │
    └───────┘                     └───────┘
        │                             │
        └─────────────────────────────┘
        需求减少 (Demand Reduction)
```

### 3.2 推力 (Push)

**定义**：让顾客对现状不满、推动他们寻求改变的力量

**类型**：
- **外部推力**：生活环境变化（如：有了第二个孩子后购物变得困难）
- **内部推力**：内在动机（如：创业者感到陷入低谷，需要激励）

**研究问题**：
- 顾客何时意识到现有方案不再有效？
- 当时他们的生活中发生了什么？

### 3.3 拉力 (Pull)

**定义**：引导顾客动机方向的吸引力

**类型**：
- **对更好生活的拉力**：顾客想象拥有解决方案后生活会如何变好
- **对特定方案的拉力**：顾客偏好某个特定产品

**研究问题**：
- 顾客调查过哪些其他产品？
- 为什么选择 X 而不是 Y？
- X 有什么是 Y 没有的？

### 3.4 惯性 (Inertia)

**定义**：阻止顾客改变、让他们保持现状的力量

**类型**：
- **选择惯性 (Habits-in-choice)**：决策时刻阻止切换的习惯
  - 例子：Excel 战胜 Lotus 1-2-3 是因为能导入/导出 Lotus 文件
- **使用惯性 (Habits-in-use)**：使用过程中的旧习惯
  - 例子：YourGrocer 顾客因习惯不提前规划饮食而回归超市购物

**应对策略**：
- 识别阻碍顾客的旧习惯
- 调整产品帮助顾客建立新习惯
- 例子：YourGrocer 发送"一键重订整箱"邮件帮助顾客规划

### 3.5 焦虑 (Anxiety)

**定义**：阻止顾客购买或使用产品的担忧

**类型**：
- **选择焦虑 (Anxiety-in-choice)**：首次使用前的未知恐惧
  - "如果用这个产品我会显得愚蠢吗？"
  - "如何付款？有保证吗？"
- **使用焦虑 (Anxiety-in-use)**：重复使用时的担忧
  - "下次还会满意吗？"
  - "我准备得够充分吗？"

**应对策略**：
- 提供试用、退款保证、折扣
- 帮助顾客可视化进步
- 提供准备指南和期望管理

## 四、两种 JTBD 解释模型

### 4.1 Jobs-As-Progress（进步视角）✅ 推荐

**倡导者**：Clayton Christensen, Bob Moesta, Alan Klement

**核心观点**：
- JTBD 是顾客试图取得的**进步**
- 关注**Be Goals**（成为什么样的人/状态）
- 任务/活动只是手段，不是目的
- 理论是描述性的（解释为什么发生），不是规定性的

**例子**：
- "被激励和启发"（Be Goal）→ 可能通过参加会议、与专家视频聊天等实现
- "成为体贴的家人"（Be Goal）→ 可能通过做饭、买酒、写诗等实现

### 4.2 Jobs-As-Activities（活动视角）

**倡导者**：Anthony Ulwick (Outcome Driven Innovation)

**核心观点**：
- JTBD 是顾客试图完成的**任务或活动**
- 关注**Do Goals**（做什么）
- 核心功能 Job 是基础，情感/社交 Job 是次要的
- 是类型学和意识形态，不是理论

**例子**：
- "钻一个 1/4 英寸的洞"
- "听音乐"
- "切割木材"

### 4.3 关键区别

| 维度 | Jobs-As-Progress | Jobs-As-Activities |
|------|------------------|-------------------|
| 关注点 | Be Goals（成为什么） | Do Goals（做什么） |
| 竞争定义 | 帮助同一进步的所有方案 | 功能相似的产品 |
| 创新方向 | 消除任务/活动 | 优化任务执行 |
| 理论性质 | 解释为什么购买 | 分类需求类型 |

## 五、JTBD 研究方法

### 5.1 访谈框架

**核心问题**：
1. "在决定用我们产品之前，你还尝试过哪些方案？"
2. "每个方案你喜欢/不喜欢什么？"
3. "如果不能再使用我们的产品，你会用什么替代？"
4. "是什么让你决定做出改变？"
5. "你当时生活中发生了什么？"

**避免的问题**：
- ❌ "如果不能再使用这个产品你会有什么感觉？"（假设性问题）
- ❌ "我们如何改进产品？"（针对产品而非 Job）
- ❌ "你对这个功能怎么看？"（确认偏误）

### 5.2 竞争分析

**识别真正竞争**：
- 顾客在解决同一 Job 时考虑的所有方案
- 包括"什么都不做"这个选项
- 包括预算来源的替代用途

**例子（Clarity）**：
- 直接竞争：LinkedIn、顾问（用股权交换）、会议
- 预算竞争：会议费用（机票、酒店、餐饮）、顾问费用
- 沉默竞争：焦虑（选择什么都不做）

### 5.3 描述 JTBD 的格式

```
更多关于：[顾客想要的进步/Be Goal]
- 情感收益
- 生活情境改善
- 自我转变

更少关于：[具体解决方案/活动]
- 产品类型
- 功能特性
- 使用场景
```

**例子（YourGrocer）**：
```
更多关于：家人有优质食物、减少购物压力、更多家庭时间、便利
更少关于：在线杂货购物/超市/本地店、支持本地社区
```

## 六、经典案例摘要

### 6.1 Clarity.fm（Dan Martell）

**Job**：创业者希望被他们尊敬的人激励和启发，以走出低谷

**关键洞察**：
- 顾客重视"信使"胜过"信息"
- 竞争包括：LinkedIn、顾问、会议、什么都不做
- 预算来自：会议预算、顾问预算，不是培训预算

**创新**：
- 定位"按需商业建议"（区别于 LinkedIn 的邮件往来）
- 强调比参加会议更便宜
- 添加准备指南减少焦虑

### 6.2 YourGrocer（Morgan Ranieri）

**Job**：家庭在不牺牲周末家庭时间的前提下获得优质食物

**关键洞察**：
- 目标客户不是"年轻城市专业人士"，而是有 2+ 孩子的家庭
- 竞争不是其他杂货配送，而是"超市购物"这个习惯
- 核心突破：消除了"优质食物 vs 家庭时间"的权衡

**创新**：
- 与本地小店合作（而非取代它们）
- 自动化邮件帮助顾客建立规划习惯
- 用店铺 Logo 传达质量（而非只用"质量"这个词）

### 6.3 chotuKool 失败案例

**问题**：Godrej 和 Clayton Christensen 为印度低收入家庭设计的低价冰箱

**失败原因**：
- 从解决方案出发（"做低价冰箱"），而非从 Job 出发
- 忽视顾客真实动机（很多人用陶罐冷却，不觉得需要冰箱）
- 错误定义竞争（认为竞争是其他冰箱，实际是与陶罐、即时购买习惯竞争）

**教训**：
- 不要从理论出发设计产品
- 不要假设低价就能创造需求
- 理解顾客如何定义竞争

## 七、实践应用指南

### 7.1 何时使用 JTBD

- 新产品/功能规划
- 市场定位和差异化
- 竞争分析
- 用户研究和访谈
- 营销信息设计
- 定价策略

### 7.2 常见错误

1. **从解决方案出发**：先决定做什么产品，再找 Job
2. **混淆人口统计与因果**：用 demographics 做产品决策
3. **只看可见数据**：过度依赖 CSAT、NPS、MAU 等指标
4. **忽视惯性和焦虑**：只关注需求生成，不关注需求减少
5. **错误定义竞争**：按产品类别而非顾客进步定义竞争

### 7.3 检查清单

**产品决策前**：
- [ ] 我们理解顾客的 Job 是什么吗？
- [ ] 我们知道顾客如何看待竞争吗？
- [ ] 我们识别了所有四力吗？
- [ ] 我们在帮助顾客进步，还是优化现有方案？

**用户访谈后**：
- [ ] 我们了解了顾客的解决方案历程吗？
- [ ] 我们发现了推力和拉力吗？
- [ ] 我们识别了惯性和焦虑吗？
- [ ] 我们能描述顾客的 Be Goal 吗？
